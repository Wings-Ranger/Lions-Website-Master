# Problems Report: 404.html

## File-specific issues

### Issue: Server configuration required for 404 routing

**Noted because:** The `404.html` page exists but requires server-side configuration to be served automatically when a visitor requests a missing URL.

**Why this matters:**
- Without server config (e.g., `.htaccess` for Apache, custom error pages for IIS/Nginx), visitors hitting a bad URL will see the server's default error page, not this styled one.

**Better way:**
Configure the hosting environment to serve `404.html` for missing routes:
- **Apache:** `ErrorDocument 404 /404.html` in `.htaccess`
- **Nginx:** `error_page 404 /404.html;` in the server block
- **GitHub Pages:** Automatically uses `404.html` if present
